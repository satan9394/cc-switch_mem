# CC Switch MEM Suite v3.17.0-mem.1

## 一眼看懂这次关键变化

- **一个安装包：** Windows x64 用户运行 `CC-Switch-MEM-Suite-v3.17.0-mem.1-Windows-x64-Setup.exe`，同时获得 CC Switch MEM 和 Claude-Mem Local `13.11.0-local.4`。
- **实时跟随模型：** Claude-Mem 使用当前 Claude Code 会话在 CC Switch 中解析出的模型；不设置固定模型，不静默回退，也不改变普通 Claude Code 请求的模型名称。
- **Token 可辨认：** 只有 Claude-Mem 后台请求标记为 `MEM`，便于在 CC Switch 消耗详情中区分；普通请求保持原名称。

## 数据安全边界

记忆、观察记录、会话转录和 SQLite 数据库保留在本地。只有 Claude-Mem 生成的脱敏摘要会通过本机回环代理，发送到用户当前明确选择的模型供应商。安装器不会上传提示词、记忆数据库、API Key、Token 或环境变量。

未找到会话模型时，请求会在本地失败，不会猜测模型或改走其他供应商。安装日志经过秘密值脱敏，并写入 `%LOCALAPPDATA%\claude-mem-local\logs\suite-install.log`。

## Windows SmartScreen

当前公开安装器没有商业 Authenticode 代码签名证书，因此 Windows SmartScreen 可能显示“未知发布者”。请只从 `satan9394/cc-switch_mem` Releases 下载，并用同一 Release 中的 `SHA256SUMS.txt` 核对文件哈希。

这是独立维护的本地安全 fork，不是上游 CC Switch 或 Claude-Mem 的官方发行版。
